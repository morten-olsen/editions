# Dialog

<div style={{ fontSize: "1.2rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, marginTop: "1rem", marginBottom: "4rem", maxWidth: "65ch" }}>
  Modal dialogs for moments that require the reader's focused attention — confirmations, destructive actions, or completion messages. The backdrop softly blurs the content behind, creating a gentle separation rather than a harsh overlay.
</div>

<div style={{ height: "1px", background: "var(--color-border)", marginBottom: "4rem" }} />

## Usage

<div style={{ display: "flex", flexDirection: "column", gap: "0.75rem", fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, maxWidth: "65ch", marginBottom: "2rem" }}>
  <div><strong style={{ color: "var(--color-ink)" }}>Confirmation</strong> — Before irreversible actions like deleting a source. Pair a destructive button with a secondary cancel.</div>
  <div><strong style={{ color: "var(--color-ink)" }}>Informational</strong> — Completion messages ("You're all caught up") that give the reader a moment of closure.</div>
  <div><strong style={{ color: "var(--color-ink)" }}>Forms</strong> — Quick edits that don't warrant a full page navigation. Keep dialog forms short — more than 3 fields probably needs its own page.</div>
</div>

<div style={{ height: "1px", background: "var(--color-border)", margin: "4rem 0" }} />

## Anatomy

<div style={{ display: "flex", flexDirection: "column", gap: "0.75rem", fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, maxWidth: "65ch" }}>
  <div><strong style={{ color: "var(--color-ink)" }}>Root</strong> — Manages open/close state.</div>
  <div><strong style={{ color: "var(--color-ink)" }}>Trigger</strong> — The element that opens the dialog. Renders as whatever element you pass via <code style={{ fontSize: "0.75rem", fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.125rem 0.375rem", borderRadius: "0.25rem" }}>render</code>.</div>
  <div><strong style={{ color: "var(--color-ink)" }}>Content</strong> — The modal panel. Centered, max-width constrained, with warm shadow and subtle border. Includes the backdrop automatically.</div>
  <div><strong style={{ color: "var(--color-ink)" }}>Title</strong> — Brief, specific. "Delete Ars Technica?" not "Are you sure?"</div>
  <div><strong style={{ color: "var(--color-ink)" }}>Description</strong> — One or two sentences explaining consequence or context.</div>
  <div><strong style={{ color: "var(--color-ink)" }}>Close</strong> — Dismisses the dialog. Wrap around a Button for styled close actions.</div>
</div>

<div style={{ height: "1px", background: "var(--color-border)", margin: "4rem 0" }} />

## Guidelines

<div style={{ display: "grid", gridTemplateColumns: "1fr 1fr", gap: "2.5rem" }}>
  <div>
    <div style={{ fontSize: "0.833rem", letterSpacing: "0.04em", textTransform: "uppercase", color: "var(--color-ink-tertiary)", marginBottom: "0.75rem" }}>Keep it brief</div>
    <div style={{ color: "var(--color-ink-secondary)", fontSize: "0.833rem", lineHeight: 1.625 }}>
      Dialogs interrupt the reading flow. Get to the point, provide clear actions, and let the reader return to what they were doing.
    </div>
  </div>
  <div>
    <div style={{ fontSize: "0.833rem", letterSpacing: "0.04em", textTransform: "uppercase", color: "var(--color-ink-tertiary)", marginBottom: "0.75rem" }}>Use sparingly</div>
    <div style={{ color: "var(--color-ink-secondary)", fontSize: "0.833rem", lineHeight: 1.625 }}>
      Most actions don't need a dialog. Inline confirmation or undo patterns are often calmer alternatives. Reserve dialogs for truly irreversible moments.
    </div>
  </div>
</div>

</div>
