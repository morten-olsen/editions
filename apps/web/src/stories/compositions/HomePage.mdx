# Home page — the magazine stand

<div style={{ fontSize: "1.2rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, marginTop: "1rem", marginBottom: "4rem", maxWidth: "65ch" }}>
  The home page is a magazine stand: each edition config gets exactly one slot, showing its latest unread issue. When you read it, the next one surfaces. No pile of unread dailies, no "5 editions waiting" guilt — just the latest of each, face-out, inviting you to pick one up if you feel like it.
</div>

## Design principles

<div style={{ display: "grid", gridTemplateColumns: "1fr 1fr", gap: "2rem", marginBottom: "4rem" }}>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>One slot per edition</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>Each edition config occupies exactly one slot on the stand, showing only the latest unread issue. Haven't checked in three days? You still see one card per config — not a backlog. Once you read the latest, the next-most-recent quietly surfaces.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Browse, don't manage</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The home page is not a dashboard, not a list of tasks. It's a storefront: visually rich, inviting, designed to make you want to pick something up and read it. Each card links to the edition itself — and to its full archive, if you're curious.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Being done is the feature</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The "all read" state uses the same ~ end mark as the magazine finale. Completion is celebrated, not treated as an empty state. The newsstand is simply closed for now.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>No anxiety, no counting</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The old design said "3 editions waiting" — a gentle guilt trip. The new design shows the stand and lets you browse. No counts, no "you have unread..." banners. If something's there, pick it up. If not, you're done.</div>
  </div>
</div>

## Data architecture

<div style={{ marginBottom: "4rem" }}>
  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, maxWidth: "65ch" }}>
    <p style={{ margin: "0 0 0.75rem" }}>The magazine stand is powered by a single <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>GET /api/home</code> endpoint. The key change: the backend now returns <strong style={{ color: "var(--color-ink)" }}>at most one edition per config</strong> — the latest unread. This is done with a SQL subquery that selects <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>MAX(published_at)</code> per <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>edition_config_id</code> where <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>read_at IS NULL</code>.</p>
    <p style={{ margin: "0 0 0.75rem" }}>The "All issues" view uses <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>GET /api/editions/configs/:configId/editions?read=false</code> with an optional <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>read</code> query parameter to toggle between unread and all issues.</p>
    <p style={{ margin: 0 }}>Article content is never fetched for either the home page or the issues list. Preview data — title, source name, image URL — is all that's needed for the cards.</p>
  </div>
</div>
