# Home page

<div style={{ fontSize: "1.2rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, marginTop: "1rem", marginBottom: "4rem", maxWidth: "65ch" }}>
  The home page is the reader's daily front page — where they arrive, where they see what's waiting, and where they feel done. It has two modes: a setup guide for new users and a reading list for configured ones. Both share a newspaper masthead that grounds the experience in time and ritual.
</div>

## Design principles

<div style={{ display: "grid", gridTemplateColumns: "1fr 1fr", gap: "2rem", marginBottom: "4rem" }}>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Calm over anxious</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>No unread counts, no urgency signals. "3 editions waiting" is information, not pressure. The reading list states a fact, it doesn't demand action.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Finite over infinite</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The "all read" state is the goal, not an absence. When you've read everything, the page celebrates completion with the same ~ end mark used in the magazine finale.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Newspaper ritual</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The masthead with today's date creates a daily rhythm — like picking up the morning paper. The mono type, uppercase label, and thin rule echo print mastheads.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Friendly over technical</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The setup guide speaks plainly. No jargon, no raw syntax. Steps explain what you'll get, not what you'll configure. The numbered steps use a familiar editorial pattern.</div>
  </div>
</div>

      {[
        { num: "01", title: "Add sources", done: true },
        { num: "02", title: "Create focuses", done: false },
        { num: "03", title: "Create an edition", done: false },
      ].map((step, i) => (

            {!step.done && (
              <div style={{ fontSize: "0.75rem", color: "var(--color-ink-tertiary)", marginTop: "0.25rem" }}>Description of what this step does…</div>

          </div>
        </div>

  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>
    <p style={{ margin: "0 0 0.75rem" }}>New users see a three-step guide styled as an editorial column, not a generic onboarding wizard. The heading "Getting started" is in serif, and the introduction explains the value proposition in one plain sentence.</p>
    <p style={{ margin: "0 0 0.75rem" }}>Steps are numbered in monospace (<strong style={{ fontFamily: "var(--font-mono)", color: "var(--color-ink)", fontSize: "0.8rem" }}>01</strong>, <strong style={{ fontFamily: "var(--font-mono)", color: "var(--color-ink)", fontSize: "0.8rem" }}>02</strong>, <strong style={{ fontFamily: "var(--font-mono)", color: "var(--color-ink)", fontSize: "0.8rem" }}>03</strong>) — a pattern borrowed from the magazine's section dividers. Completed steps show an accent-colored check mark and muted title, giving a satisfying sense of progress.</p>
    <p style={{ margin: "0 0 0.75rem" }}>Each uncompleted step has a primary CTA button. The button text is action-specific ("Add your first source", not "Get started") to be clear about what clicking it will do.</p>
    <p style={{ margin: 0 }}>Once the user has added at least one source, a subtle "Browse all articles →" link appears below the steps — an escape hatch for readers who want to explore before finishing setup.</p>
  </div>

  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>
    <p style={{ margin: "0 0 0.75rem" }}>Each unread edition appears as a newspaper-style teaser. The layout has three layers of typography working together:</p>
    <p style={{ margin: "0 0 0.75rem" }}><strong style={{ color: "var(--color-ink)" }}>Mono (wayfinding)</strong> — the edition config name and date in small uppercase monospace, in accent color. This tells you <em>which</em> edition and <em>when</em>. The same mono-for-navigation pattern appears throughout the magazine view.</p>
    <p style={{ margin: "0 0 0.75rem" }}><strong style={{ color: "var(--color-ink)" }}>Serif (content)</strong> — the edition title in large serif. This is the headline — the thing you'll be reading. It uses the same font and weight as the magazine cover headline, creating visual continuity.</p>
    <p style={{ margin: "0 0 0.75rem" }}><strong style={{ color: "var(--color-ink)" }}>Mono (metadata)</strong> — article count and reading time in small monospace, faint. A commitment, not a claim — the same pattern as the magazine finale's stats.</p>
    <p style={{ margin: 0 }}>On hover, the top border turns accent and the title shifts to accent color. A small chevron arrow at the right nudges 2px rightward — a micro-interaction that signals navigability without demanding attention.</p>

## The "all read" state

<div style={{ display: "flex", gap: "4rem", marginBottom: "4rem", alignItems: "flex-start" }}>
  <div style={{ width: "20rem", flexShrink: 0 }}>
    <div style={{ border: "1px solid var(--color-border)", borderRadius: "0.5rem", background: "var(--color-surface)", padding: "2rem 1.5rem", textAlign: "center" }}>
      <div style={{ fontSize: "2.5rem", color: "var(--color-accent)", opacity: 0.2, marginBottom: "0.75rem", userSelect: "none" }}>~</div>
      <div style={{ fontFamily: "var(--font-serif)", fontSize: "1.25rem", fontWeight: 500, letterSpacing: "-0.02em", color: "var(--color-ink)", marginBottom: "0.375rem" }}>All read</div>
      <div style={{ fontSize: "0.833rem", color: "var(--color-ink-tertiary)", lineHeight: 1.625 }}>You're up to date. New editions will appear here when they're generated.</div>
    </div>
  </div>
  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>
    <p style={{ margin: "0 0 0.75rem" }}>This is the home page's most important state. When there's nothing left to read, the page doesn't feel empty — it feels <em>complete</em>. The design borrows from the magazine finale: the same <strong style={{ color: "var(--color-ink)" }}>~</strong> end mark that signals "you're done" at the end of an edition appears here too.</p>
    <p style={{ margin: "0 0 0.75rem" }}>The tilde is a typographic convention from newspaper journalism — the "30" or "###" that historically marked the end of a piece. Here it appears in accent color at 20% opacity, large and centered. It's decorative, not informational.</p>
    <p style={{ margin: 0 }}>The heading says "All read" — two words, factual, satisfied. Not "You're all caught up!" (too excited), not "No unread items" (too technical). The supporting text explains what happens next without creating urgency.</p>
  </div>
</div>

  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>
    <p style={{ margin: "0 0 0.75rem" }}>Below the reading list (or empty state), a row of quick links provides shortcuts to the feed, bookmarks, and each edition config. These are text-only — no icons, no borders, no boxes. Just small monospace labels that turn from tertiary to primary ink on hover.</p>
    <p style={{ margin: 0 }}>The links are deliberately understated. The reading list is the main event; quick links are a convenience for power users who want to navigate without using the sidebar. Edition config links include an entity icon when available, maintaining the same visual language as the sidebar nav.</p>
