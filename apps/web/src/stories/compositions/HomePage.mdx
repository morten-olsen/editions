# Home page — the newsstand

<div style={{ fontSize: "1.2rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, marginTop: "1rem", marginBottom: "4rem", maxWidth: "65ch" }}>
  If the magazine view is settling into a favourite chair to read, the home page is browsing the newsstand — seeing what's waiting for you, picking up the one that catches your eye. Each unread edition is displayed as a magazine cover, with the lead story's image, section tags, and reading time creating the visual pull of a real storefront.
</div>

## Design principles

<div style={{ display: "grid", gridTemplateColumns: "1fr 1fr", gap: "2rem", marginBottom: "4rem" }}>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Browse, don't manage</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The home page is not a dashboard, not a list of tasks. It's a storefront: visually rich, inviting, designed to make you want to pick something up and read it.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Being done is the feature</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The "all read" state uses the same ~ end mark as the magazine finale. Completion is celebrated, not treated as an empty state. The newsstand is simply closed for now.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Visual hierarchy</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The most recent edition gets the full cover card treatment — image background, large headline, section tags. Older editions become compact teasers. A real newsstand puts the latest issue face-out.</div>
  </div>
  <div style={{ padding: "1.25rem", borderLeft: "2px solid var(--color-accent)", background: "var(--color-surface)" }}>
    <div style={{ fontSize: "0.833rem", fontWeight: 600, color: "var(--color-ink)", marginBottom: "0.5rem" }}>Single request</div>
    <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>The <code style={{ fontFamily: "var(--font-mono)", fontSize: "0.8rem" }}>GET /api/home</code> endpoint returns everything in one call: counts, configs, and edition previews with lead article data. No N+1 queries, no waterfalls.</div>
  </div>
</div>

## Data architecture

<div style={{ marginBottom: "4rem" }}>
  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, maxWidth: "65ch" }}>
    <p style={{ margin: "0 0 0.75rem" }}>The newsstand is powered by a single <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>GET /api/home</code> endpoint that returns everything needed in one request:</p>
  </div>

  <div style={{ fontFamily: "var(--font-mono)", fontSize: "0.75rem", lineHeight: 1.6, background: "var(--color-surface-sunken)", padding: "1.25rem", borderRadius: "0.5rem", color: "var(--color-ink-secondary)", marginBottom: "1rem", maxWidth: "40rem" }}>
    <div>{"{"}</div>
    <div style={{ paddingLeft: "1rem" }}>sourcesCount: number,</div>
    <div style={{ paddingLeft: "1rem" }}>focusesCount: number,</div>
    <div style={{ paddingLeft: "1rem" }}>configs: [{"{ id, name, icon }"}],</div>
    <div style={{ paddingLeft: "1rem" }}>editions: [{"{"}</div>
    <div style={{ paddingLeft: "2rem" }}>id, title, configName, configIcon,</div>
    <div style={{ paddingLeft: "2rem" }}>articleCount, totalReadingMinutes,</div>
    <div style={{ paddingLeft: "2rem" }}>sections: [{"{ focusName, articleCount }"}],</div>
    <div style={{ paddingLeft: "2rem" }}>lead: {"{ title, sourceName, imageUrl }"},</div>
    <div style={{ paddingLeft: "2rem" }}>highlights: [{"{ title, sourceName }"}]</div>
    <div style={{ paddingLeft: "1rem" }}>{"}"}]</div>
    <div>{"}"}</div>
  </div>

  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, maxWidth: "65ch" }}>
    <p style={{ margin: "0 0 0.75rem" }}>The lead article is selected by scanning edition articles in position order for the first one with an image. If none have images, position 0 is used regardless. This maximises the chance of a visually rich cover card.</p>
    <p style={{ margin: 0 }}>Article content (the heavy HTML body) is never fetched for the home page. The preview data — title, source name, image URL — is all that's needed for the newsstand cards.</p>

## Setup guide

<div style={{ display: "flex", gap: "4rem", marginBottom: "4rem", alignItems: "flex-start" }}>
  <div style={{ width: "20rem", flexShrink: 0 }}>
    <div style={{ border: "1px solid var(--color-border)", borderRadius: "0.5rem", background: "var(--color-surface)", padding: "1.25rem" }}>
      <div style={{ fontFamily: "var(--font-serif)", fontSize: "1.4rem", letterSpacing: "-0.02em", color: "var(--color-ink)", marginBottom: "0.5rem" }}>Getting started</div>
      <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, marginBottom: "1.25rem" }}>A finite, curated reading experience.</div>

      {[
        { num: "01", title: "Add sources", done: true },
        { num: "02", title: "Create focuses", done: false },
        { num: "03", title: "Create an edition", done: false },
      ].map((step, i) => (
        <div key={i} style={{ padding: "0.75rem 0", borderTop: "1px solid var(--color-border)", display: "flex", alignItems: "flex-start", gap: "0.75rem" }}>
          <div style={{ width: "1.25rem", flexShrink: 0, paddingTop: "0.125rem" }}>
            {step.done ? (
              <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 16 16" fill="currentColor" style={{ width: "0.875rem", height: "0.875rem", color: "var(--color-accent)" }}>
                <path fillRule="evenodd" d="M12.416 3.376a.75.75 0 0 1 .208 1.04l-5 7.5a.75.75 0 0 1-1.154.114l-3-3a.75.75 0 0 1 1.06-1.06l2.353 2.353 4.493-6.74a.75.75 0 0 1 1.04-.207Z" clipRule="evenodd" />
              </svg>
            ) : (
              <span style={{ fontFamily: "var(--font-mono)", fontSize: "0.833rem", color: "var(--color-ink-faint)" }}>{step.num}</span>

          <div style={{ fontFamily: "var(--font-serif)", fontSize: "1rem", fontWeight: 500, letterSpacing: "-0.02em", color: step.done ? "var(--color-ink-tertiary)" : "var(--color-ink)" }}>{step.title}</div>
        </div>
      ))}
      <div style={{ height: "1px", background: "var(--color-border)" }} />
    </div>
  </div>
  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>
    <p style={{ margin: "0 0 0.75rem" }}>New users see a three-step guide before the newsstand appears. Steps are numbered in monospace (<strong style={{ fontFamily: "var(--font-mono)", color: "var(--color-ink)", fontSize: "0.8rem" }}>01</strong>, <strong style={{ fontFamily: "var(--font-mono)", color: "var(--color-ink)", fontSize: "0.8rem" }}>02</strong>, <strong style={{ fontFamily: "var(--font-mono)", color: "var(--color-ink)", fontSize: "0.8rem" }}>03</strong>) — matching the magazine section divider pattern.</p>
    <p style={{ margin: "0 0 0.75rem" }}>Completed steps show an accent check mark. Steps are sequential — only the next uncompleted step shows a CTA button. Future steps are dimmed (faint text, no button) until their prerequisites are met: sources before focuses, focuses before editions.</p>
    <p style={{ margin: 0 }}>The setup guide is shown when any of the three prerequisites (sources, focuses, edition configs) are missing. Once all three exist, the newsstand takes over.</p>
