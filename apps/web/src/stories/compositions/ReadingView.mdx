# Reading view

<div style={{ fontSize: "1.2rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, marginTop: "1rem", marginBottom: "4rem", maxWidth: "65ch" }}>
  The reading view is for reading a single article in full. It's calmer and simpler than the magazine — a long prose column, minimal chrome, no pagination. The interface gets out of the way.
</div>

  <div style={{ width: "20rem", flexShrink: 0 }}>

          omething is shifting in the way we consume the written word online. After years of infinite scrolling, a new generation is rethinking everything.
        </p>
        <p style={{ margin: "0 0 0.875rem", color: "var(--color-ink-secondary)" }}>
          These apps don't try to keep you engaged for as long as possible. They present articles in a way that respects your time and then let you go.
        </p>
        <h2 style={{ fontFamily: "var(--font-serif)", fontSize: "1.1rem", fontWeight: 500, letterSpacing: "-0.01em", color: "var(--color-ink)", margin: "1.25rem 0 0.625rem" }}>
          Typography as interface
        </h2>
        <p style={{ margin: "0 0 0.875rem", color: "var(--color-ink-secondary)" }}>
          One pattern emerging is typography as the primary interface element. A serif signals "this is content". A sans-serif signals "this is interface".
        </p>
        <blockquote style={{ borderLeft: "2px solid var(--color-accent)", paddingLeft: "1rem", margin: "1rem 0", fontStyle: "normal", color: "var(--color-ink-secondary)", fontSize: "0.95rem" }}>
          "The best reading experience is one where you forget you're using an app at all."
        </blockquote>
      </div>
    </div>
  </div>
  <div style={{ fontSize: "0.833rem", color: "var(--color-ink-secondary)", lineHeight: 1.625 }}>
    <p style={{ margin: "0 0 0.75rem" }}>The prose column uses Tailwind's <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>prose</code> typography plugin, tuned with our own color tokens. Headings, paragraphs, blockquotes, links, and code all inherit the design system's palette.</p>
    <p style={{ margin: "0 0 0.75rem" }}>The <strong style={{ color: "var(--color-ink)" }}>drop cap</strong> on the first paragraph's opening letter is the same treatment as the magazine article body — a typographic tradition that signals "start reading here". Applied via CSS <code style={{ fontFamily: "var(--font-mono)", background: "var(--color-surface-sunken)", padding: "0.1rem 0.3rem", borderRadius: "0.2rem", fontSize: "0.8rem" }}>first-letter</code> selector, it only appears on the very first paragraph.</p>
    <p style={{ margin: "0 0 0.75rem" }}><strong style={{ color: "var(--color-ink)" }}>Dual typography</strong> — serif for content, sans-serif for interface elements — runs through the whole design system. In the reading view it's most visible: the article body is serif, the header and meta information are sans. Readers unconsciously learn the distinction and stop "seeing" the chrome.</p>
    <p style={{ margin: 0 }}>Blockquotes use the same accent left border as the magazine: no italics, no indentation, just the color signal. The accent border is consistent across the whole system, so blockquotes always feel the same regardless of which surface they appear on.</p>

    <p style={{ margin: 0 }}>There is no "related articles" section. No autoplay. No recommendation carousel. The end of the article is the end. This is intentional and non-negotiable — it's what makes the reading experience feel respectful rather than extractive.</p>
