# Typography

<div style={{ fontSize: "1.2rem", color: "var(--color-ink-secondary)", lineHeight: 1.625, marginTop: "1rem", marginBottom: "4rem", maxWidth: "65ch" }}>
  Typography is our primary design material. Two families — a serif for reading, a sans-serif for interface — work in concert to create a clear hierarchy between content and chrome. The type scale follows a minor third ratio for gentle, natural progression.
</div>

<div style={{ height: "1px", background: "var(--color-border)", marginBottom: "4rem" }} />

## Families

<div style={{ color: "var(--color-ink-secondary)", fontSize: "0.833rem", marginBottom: "2.5rem", maxWidth: "65ch" }}>
  The serif carries the editorial voice — warm, authoritative, inviting for extended reading. The sans-serif is clean and functional, stepping back so the content can speak. Monospace appears only where precision matters.
</div>

## The Dual System

<div style={{ color: "var(--color-ink-secondary)", fontSize: "0.833rem", marginBottom: "2.5rem", maxWidth: "65ch" }}>
  The boundary between serif and sans isn't decorative — it signals where the interface ends and the content begins. Serif says "this is something worth reading." Sans says "this is a tool to help you navigate." When both appear in the same view, the reader instinctively knows which is chrome and which is substance.
</div>

<DualSystem />

<div style={{ height: "1px", background: "var(--color-border)", margin: "4rem 0" }} />

## Scale

<div style={{ color: "var(--color-ink-secondary)", fontSize: "0.833rem", marginBottom: "2.5rem", maxWidth: "65ch" }}>
  Built on a minor third ratio (1.2). Each step feels like a natural increment, never a jarring jump. Display sizes use tighter tracking; body text breathes with generous line heights.
</div>

<TypeScale />

<div style={{ height: "1px", background: "var(--color-border)", margin: "4rem 0" }} />

## Measure

<div style={{ color: "var(--color-ink-secondary)", fontSize: "0.833rem", marginBottom: "2.5rem", maxWidth: "65ch" }}>
  Comfortable reading happens between 45 and 75 characters per line. Our prose width of 65ch sits in the sweet spot. This constraint isn't a limitation — it's where the reading happens.
</div>
